# Shikki Promo Video — First Draft Storyboard

> 60-90 seconds. No voiceover (yet). Music + SFX + text only.
> Target: GitHub README hero, Twitter/X, Product Hunt launch.

---

## Shot 1: The World Before (3s)

**Visual**: Normal environment — desk, coffee, nature through window. Calm.
**Device**: A MacBook (or iPad, or Apple Vision Pro) sits on the desk.
**Feel**: Quiet. Everyday. Nothing special yet.
**SFX**: Ambient room tone.

---

## Shot 2: Terminal Awakens (2s)

**Visual**: On the device screen, from the bottom, a terminal window bounces up with a spring animation. Starts at alpha 0, lands center-screen.
**Style**: Dark terminal, sleek border, minimal. The Replaced/Last Night pixel aesthetic.
**SFX**: Soft "pop" + glass tap.

---

## Shot 3: The Birth (4s)

**Visual**: Sparkles appear around the terminal. Noise particles. Abstract shapes — lightning bolts, thought bubbles, gears. The screen flickers slightly.
**Center**: A tiny blue flame starts to form. Pixel by pixel. First a dot, then a flicker, then the full Blue Flame with manga eyes.
**SFX**: Building energy — rising synth tone + sparkle sounds.
**Text overlay** (subtle, bottom): *"What if your ideas could build themselves?"*

---

## Shot 4: Shikki Prompt (3s)

**Visual**: The flame settles in the terminal. A prompt appears:

```
shi › build me a todo app with auth ▊
```

The text types character by character, like a real terminal input. The cursor blinks.
**SFX**: Keyboard typing sounds (mechanical, satisfying).

---

## Shot 5: The Dispatch (5s)

**Visual**: The main terminal shrinks to the left. Three new terminals pop open — 1, 2, 3 — with spring animations, each slightly offset.

Each terminal shows:
- Terminal 1: Code scrolling fast (Swift/TS — real code, not gibberish)
- Terminal 2: Different code scrolling — tests, models
- Terminal 3: More code — UI components, API routes

**Style**: Hollywood hacking montage but real. The code is actual Shikki output — not fake.
**SFX**: Triple "pop" for each terminal + fast typing ambient + electronic pulse.

---

## Shot 6: First Green (2s)

**Visual**: Terminal 1 stops scrolling. A green checkmark appears:

```
✅ Auth module — 12 tests passing
```

The terminal glows green for a beat, then smoothly closes — shrinking to nothing.
Terminals 2 and 3 rearrange, taking the freed space.
**SFX**: Success chime (short, satisfying).

---

## Shot 7: Cascade Complete (3s)

**Visual**: Less than a second later, Terminal 2 shows its green check:

```
✅ API routes — 8 tests passing
```

It closes. Terminal 3 rearranges to center, then also completes:

```
✅ UI components — 15 tests passing
```

It closes. The main Shikki terminal is alone again, center screen.
**SFX**: Three rapid success chimes in cascade — da-da-da.

---

## Shot 8: Review Time (4s)

**Visual**: On the Shikki terminal, new text appears:

```
📦 App ready for review

  Take a look? [Y/n] ▊
```

The cursor blinks. A moment of pause (the user decides).
Then "y" types. Enter.

**SFX**: Single keystroke. Beat of silence. Then...

---

## Shot 9: The Review Panel (4s)

**Visual**: A panel slides up from the bottom of the terminal — 3 lines:

```
  ✅ Auth — secure, JWT tokens, session management
  ❌ API — missing rate limiting on /login endpoint
  ✅ UI — responsive, accessible, dark mode support
```

Below it:

```
What's next? ▊
```

Cursor blinks. The user types:

```
Fix 2.
```

**SFX**: Subtle alert sound on the ❌, then typing.

---

## Shot 10: The Fix (3s)

**Visual**: A single new terminal pops to the right. Code scrolls — focused, smaller, faster than before. It knows exactly what to fix.

```
✅ Rate limiter added — 3 tests passing
```

Terminal closes. Back on Shikki terminal:

```
  ✅ Auth — secure, JWT tokens, session management
  ✅ API — rate limiting added, all endpoints protected  ← was ❌
  ✅ UI — responsive, accessible, dark mode support
```

The ❌ becomes ✅ with a smooth animation.

**SFX**: Fix sound + success chime.

---

## Shot 11: Ship It (3s)

**Visual**: A large button appears at the bottom of the terminal:

```
┌─────────────────────────────┐
│         🚀 SHIP IT          │
└─────────────────────────────┘
```

The user hits Enter. The button pulses. The Blue Flame in the corner does a little celebration dance (eyes close, sparkles).

**SFX**: Satisfying "launch" sound — deep bass + rising tone.

---

## Shot 12: The Tagline (4s)

**Visual**: The terminal fades. The Blue Flame floats to center screen, large now. Calm. Breathing.

Text appears below:

```
Shikki
━━━━━━
The engine that builds while you think.
```

Then smaller:

```
shikki.dev
```

**SFX**: Music resolves. Silence. The flame flickers gently.

---

## Shot 13: The Counter (2s)

**Visual**: Quick stats fade in around the flame:

```
                    35 tests ✅
        3 agents    1 spec      < 60 seconds
```

**SFX**: None. Let it breathe.

## Production Notes

### Music
Lo-fi electronic, building energy. Think: Tycho meets Trent Reznor. Starts ambient, builds during dispatch, resolves on ship.

### Visual Style
- Terminal: dark background, monospace, real code
- Blue Flame: pixel art, 2.5D, Replaced/Last Night aesthetic
- Environment: minimal, clean, focus on the screen
- Color palette: dark grays + cyan/teal accents + green for success + red for failure

### What We Need
1. Blue Flame animation sprites (calm, working, celebrating) — from Procreate sketches
2. Terminal recording (real Shikki session, scripted)
3. Motion graphics for sparkles/dispatch/transitions
4. Music track (commission or license)
5. Sound design (keyboard, chimes, whoosh)

### Tools
- Terminal recording: `vhs` (charm.sh) or `asciinema`
- Motion graphics: After Effects or Motion (macOS)
- Compositing: Final Cut Pro or DaVinci Resolve
- Pixel art animation: Procreate + Aseprite

### What This Video Does NOT Show
- No pricing
- No feature list
- No comparison to competitors
- No voiceover explaining what Shikki is
- Just: idea → agents → tests → review → fix → ship. In 60 seconds.
- The viewer FEELS it before they understand it.
