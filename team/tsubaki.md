## Cross-Project Learnings

### Copy Patterns

- **Hero sections**: Lead with emotion, follow with function. "Embrace your imperfect practice" before "Track habits mindfully."
- **Onboarding**: One concept per screen. Short, memorable phrases. Animation carries the narrative — text supports, doesn't lead.
- **App Store descriptions**: First line is the hook. Feature list as benefits, not specs. Social proof if available.
- **SEO**: Natural keyword integration. Include common misspellings in meta tags. FAQ schema for featured snippets.

### Content Architecture

- **JSON-LD structured data**: @graph with multiple entities (SoftwareApplication, FAQPage, Organization, WebSite). Rich search results.
- **Smart Banner**: iOS meta tag for App Store link. Unobtrusive, native feel.
- **Deep links**: `wabisabi://` protocol for landing page → app handoff. Footer zen input creates todo via deep link.

### Tone Calibration

- **Mindfulness product**: Quieter, more spacious. Shorter sentences. More breathing room.
- **Dev tool (Shiki)**: Technical precision. Direct. No fluff. Developers respect conciseness.
- **Design system (DSKintsugi)**: Documentation tone — clear, scannable, example-driven.

### Anti-Patterns Observed

- Writing for algorithms instead of humans
- Feature-first copy (listing features before establishing emotional connection)
- Generic placeholder copy that never gets replaced ("Lorem ipsum" syndrome)
- Inconsistent tone across app / website / store listing
