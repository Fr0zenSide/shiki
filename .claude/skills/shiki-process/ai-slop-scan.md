# Pre-Release AI Slop Scan

Scan shipped source code for AI-generated markers before release. Zero tolerance.

## When to run

- Before every release merge (e.g., `develop → main`)
- As Gate 8 in the `/pre-pr` pipeline (release PRs only)
- On demand via `/pre-release-scan`

## Patterns to scan for

### 1. Agent mentions in source code
```
@Sensei, @Hanami, @Kintsugi, @Enso, @Tsubaki, @Shogun, @Daimyo, @tech-expert
```

### 2. AI attribution
```
Co-Authored-By: Claude
Generated with Claude
Generated by AI
AI-generated
```

### 3. Tool markers
```
Claude Code, Anthropic, GPT-, Copilot, AI assistant, AI-assisted
```

### 4. Process artifacts
```
/md-feature, /pre-pr, shiki-process, agent orchestration
```

### 5. TODO/FIXME with agent names
```
// TODO(@Sensei), // FIXME(@Hanami), // reviewed by @Kintsugi
```

## Scan command

Adapt to your project's source directories (from project adapter). Example for a multi-language project:

```bash
rg -i "(co-authored-by.*claude|generated.*claude|generated.*ai|ai.generated|@sensei|@hanami|@kintsugi|@enso|@tsubaki|@shogun|@daimyo|@tech-expert|claude.code|anthropic|gpt-|copilot|ai.assistant|ai.assisted|/md-feature|/pre-pr|shiki.process)" \
  -g '!.claude/' \
  -g '!*.md' \
  -g '!features/' \
  -g '!node_modules/' \
  -g '!.git/' \
  {source_directories}
```

If a `project-adapter.md` exists, read it for:
- `source_directories` — where shipped code lives
- `file_types` — which file types to scan (e.g., `--type swift`, `--type ts`)
- `exclusions` — additional directories to exclude

## Exclusions (not shipped)

| Directory | Reason |
|-----------|--------|
| `.claude/` | Claude Code tooling config |
| `features/` | Feature documentation |
| `*.md` | Markdown docs (not compiled) |
| `node_modules/` | Third-party dependencies |

Add project-specific exclusions from the project adapter.

## Also check

- **Git commit messages**: Co-Authored-By tags are OK in commits (users can't see those), but double-check no commit message ends up in compiled code
- **Config files**: No AI references in app metadata (Info.plist, package.json description, etc.)
- **Localization files**: `.strings`, `.xcstrings`, `.json` i18n files — no agent names
- **Asset catalogs**: No AI-related asset names

## Output

### CLEAN
```markdown
## AI Slop Scan
- Status: CLEAN
- Files scanned: 247
- Matches: 0
```

### FAIL
```markdown
## AI Slop Scan
- Status: FAIL
- Matches: 3
  - src/services/HabitService.ts:42 — "// @Sensei reviewed"
  - src/config.ts:15 — "// Generated with Claude Code"
  - tests/HabitTests.ts:8 — "// co-authored-by Claude"
- Action: Remove these references before release
```

## Remediation

When matches are found:
1. Remove the reference entirely (preferred)
2. Or replace with neutral text (e.g., "// Code review passed" instead of "// @Sensei reviewed")
3. Re-run scan to confirm CLEAN
4. Proceed with release

## Anti-Rationalization

| Thought | Response |
|---------|----------|
| "This comment is harmless, no one will notice" | Automated review tools see everything. Zero tolerance means zero exceptions. |
| "It's in a test file, not production code" | Test files may ship in test bundles. Remove it. |
| "The reference is in a private method name, no user will see it" | Code analysis tools see everything. Remove it. |
| "I'll fix it later before the actual release" | Fix it now. 'Later' means 'forgotten'. The scan exists to catch it NOW. |
| "Co-Authored-By is standard, users can't see git history" | Correct — git history is safe. But verify the reference isn't in compiled code or strings files. |
| "This is a false positive, the word 'copilot' is part of our feature name" | Rename the feature. Don't risk issues over a name collision with an AI tool. |
